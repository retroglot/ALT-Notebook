# Ⅰ 引言

## 一、架构原则

### 1. 数据包、连接和数据报

**电话网络｜Telephone Network**　网络概念的起源。电话网络从模拟基础发展为数字化产品，数据沿着 *网络交换机* 设定的路径传输，从电路一端输入，另一端延迟输出。

**分组交换｜Packet Switching**　若干数字信息组成“块”（分组），在网络中相对独立传输。分组可以混合在一起，随后被拆分，在前往目的地的过程中灵活拆分，并在各个交换节点动态改变传输路径。

**多路复用｜Multiplexing**　通过分组交换混合复用节点和链路，减少物理攻击影响、增加链路和节点的利用率。

**分组交换的实现方式**　当数据包到达分组交换机时，被存储在 buffer memory / queue 中，并以 *FCFS-先到先服务* 的方式处理，被称为 *FIFO-先进先出* 。

**统计复用｜Implement Stastical Multiplexing**　互联网上混合来自不同源的流量的主要方法。流量根据其到达的统计特性或时间模式进行混合，因而简单高效。然而，统计复用法的缺点是可预测性有限，某个应用的流量可能被其他共享网络应用的流量大幅影响。

**静态复用｜Static Multiplexing**　相对于统计复用的替代技术。通过预先分配固定的资源给每个连接或数据流，而不考虑其实际流量需求。

**时分复用｜Time-Division Multiplexing**　一种静态复用技术，又称 TDM 。为每条连接上的数据预留一定的时间，但预留的带宽会被浪费。

**虚拟电路｜Virtual Circuits**　又称 VC 。指数据包分组交换网络上，基于统计复用提供面向连接，并通过软件/协议模拟出静态复用的效果。在虚拟电路的基础上发展出了 *X.25协议* 。

后来，VC 被 帧中继技术 取代，最终又被 DSL 和 调制解调器 取代。

**面向连接｜Connection-Oriented**　每个交换机为每条连接存储某些信息/状态，用作状态表的索引。在 *X.25协议* 中，采用的是12位的 *逻辑信道标识符-LCI* 或 *逻辑信道号-LCN* 在每个交换机中与每条流的状态相结合，以确定数据包沿路径的下一跳交换机。

---

**数据报｜Datagram**　一种特殊的分组，源地址和最终目的地址的所有标识信息均包含在分组自身内部，而非分组交换机。

- 需要更大的分组；
- 消除了分组交换机对每条连接状态的维护需求

数据报方案避免了使用复杂信令协议的必要性，很快被早期互联网的设计者们采纳。
